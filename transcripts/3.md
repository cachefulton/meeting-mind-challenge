That's the global **NestJS ValidationPipe** we just added. It automatically validates every incoming request body against the DTO decorators (like the `@IsString()`, `@IsNotEmpty()`, `@IsDateString()` in `create-meeting.dto.ts`). Here's what each option does:

- **`whitelist: true`** тАФ Strips any properties from the request body that don't have a decorator in the DTO. So if someone sends `{ title: "...", hackerField: "gotcha" }`, the `hackerField` is silently removed before it reaches your controller.

- **`forbidNonWhitelisted: true`** тАФ Goes a step further than `whitelist`: instead of silently stripping unknown properties, it **rejects the request** with a 400 error. This makes the API strict тАФ clients get a clear error if they send unexpected fields.

- **`transform: true`** тАФ Automatically transforms the plain JSON object from the request into an instance of the DTO class. Without this, `@Body() dto` would be a plain object and `instanceof CreateMeetingDto` would be `false`. With it, class-transformer instantiates the class, which is required for class-validator decorators to work properly.

In short: any `POST /meetings` request that doesn't match the shape defined in `CreateMeetingDto` gets rejected with a descriptive 400 error before your controller code ever runs.
